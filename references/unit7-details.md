# 单元七详细内容：目标、决策与执行

> 从"带人"到"把事做成"。

## OKR 目标管理
*来源：Andy Grove（Intel）→ John Doerr（Google）*

- **O 目标（Objective）**：鼓舞人心、定性、有方向感
- **KR 关键结果（Key Results）**：可量化、3–5 个、用来衡量 O 是否达成
- 公司 O → 团队 O → 个人 O，上下对齐贯通

OKR ≠ KPI：

| | OKR | KPI |
|---|-----|-----|
| 导向 | 方向、挑战、对齐 | 考核、达标 |
| 满分 | 0.7 即优秀（敢定高目标） | 必须 100% 完成 |
| 用途 | 聚焦最重要的事 | 绩效衡量 |

> 💼 **职场案例**：你给团队定目标"提升用户体验"，结果三个月后谁也说不清做到没做到。
> **→ 改法**：O 保留"打造行业最佳用户体验"，但配上可量化 KR——"NPS 从 30→45 / 页面加载 <2s / 客诉率降 50%"。**目标要能被衡量，否则就是口号。**

> ⚠️ **反模式**：① 把 OKR 当 KPI 考核 → 逼大家把目标定保守；② KR 不可量化（"加强""提升"）；③ 一个周期定太多 O = 没重点。

## 决策与分工：RACI / RAPID
*来源：RACI 项目管理通用；RAPID 来自 Bain*

专治"决策卡壳、互相扯皮、谁都能否决却没人拍板"。核心是**明确角色，尤其唯一的拍板人**。
- **RACI**（任务分工）：Responsible 执行（可多个）· **Accountable 拍板（必须唯一）** · Consulted 事先咨询（双向）· Informed 事后知会（单向）
- **RAPID**（决策版）：Recommend 提建议 · Agree 须同意 · Perform 执行 · Input 提供输入 · **Decide 拍板（唯一）**

> 💼 **职场案例**：一个跨部门项目，三个部门谁都能提反对意见、谁都不肯拍板，拖了一个月没进展。
> **→ 行动**：用 RAPID 先明确**唯一的 Decider**（拍板人），其他部门是 Input/Agree 角色。决策权一清晰，扯皮立刻减少。

> ⚠️ **反模式**：Accountable / Decide 出现多个人 = 没人真正负责。一件事的拍板人必须唯一。

## 艾森豪威尔矩阵（时间与精力管理）

按**重要 × 紧急**两维分四象限：
- 重要且紧急 → **立刻做**（危机、救火、deadline）
- **重要不紧急 → 计划做 ⭐**（战略、培养人、建机制）
- 不重要但紧急 → **授权出去**（打断、部分会议、杂务）
- 都不重要 → **删除/少做**（刷手机、无效社交）

管理者最该刻意投资"重要不紧急"象限，但它最容易被"紧急"挤掉。

> 💼 **职场案例**：你每天忙得团团转，全在救火（紧急），但团队能力建设、流程优化这些事永远"没空做"。
> **→ 行动**：每周固定划出时间块给"重要不紧急"象限（哪怕 2 小时）。**管理者的价值恰在第二象限——不投资它，你很可能长期陷在救火中。**

> ⚠️ **反模式**：被"紧急"绑架，把时间全花在第一、三象限，忽视真正创造长期价值的"重要不紧急"。
