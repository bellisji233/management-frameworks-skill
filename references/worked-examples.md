# 黄金范例：完整的"问题 → 诊断 → 行动"示范

> 这些是**输出标准的标杆**。回答管理决策类问题时，结构、颗粒度、语气都对齐这里。
> 核心要求：先诊断问题类型，再匹配 1–2 个框架（不要堆框架），最后给 3 步内可落地的行动。

---

## 范例 1 · 个人管理：高绩效老员工开始"躺平"

**用户问**：我手下有个老员工，能力很强，以前很拼，最近半年明显躺平了，交代的事按时做完就走，多一点不愿意干。怎么办？

**✅ 标准输出：**

**一、先定位（不要急着激励）**
这是典型的「高能力 + 低意愿」象限（情境领导力矩阵 ⑨ 区）。
⚠️ 关键判断：**他不是不会，是不想**。所以一切"教方法、给资源"的动作都无效——问题在动机，不在能力。

**二、诊断动机断点（期望理论 · Vroom）**
躺平 = 努力→绩效→奖励 这条链断了某一环。先通过一次 1:1 搞清楚断在哪：
- **断点1·能力感知**："再努力也没用了"——通常不是他的情况（他能力强）
- **断点2·兑现信任**："拼了也没回报"——老员工最常见。是不是晋升/加薪没兑现过？
- **断点3·价值契合**："要的东西这里给不了"——职业天花板，看不到新意义

**三、3 步行动**
1. **开一次"诊断式" 1:1**：不谈业绩，问感受。"我注意到你状态和以前不太一样，是遇到什么了吗？"——先听，别先劝。
2. **按断点对症**：
   - 若是"兑现信任"断了 → 给一个明确的、有时间点的兑现承诺（晋升路径 / 项目主导权），并说到做到
   - 若是"价值契合"断了 → 给新意义：带新人、跨部门项目、接班人培养（情境矩阵 ⑧ 给予挑战）
3. **30 天后复查**：看主动性行为有没有回来（是否开始管职责外的事）。没有，就升级为 ⑨ 警示对话——能力强的人躺平传染性最强，不能无限等。

**理论来源**：情境领导力（Hersey & Blanchard, 1969）、期望理论（Vroom, 1964）。

## 范例 2 · 组织管理：上级觉得"你们团队没产出"

**用户问**：我带技术团队，明明做了很多基础建设和优化，但老板总觉得我们没产出、动作慢，每次汇报都很被动。怎么破？

**✅ 标准输出：**

**一、先拆问题（这其实是两个问题，别混着解）**
- **问题A·认知/信息**：老板看不见你做的事（基础建设天然"隐形"）→ 这是信息不对称
- **问题B·资源/能力**：是否真有执行瓶颈拖慢了交付？

先诚实自评：如果产出其实在线，只是没被看见 → 主攻 A；如果确实慢了 → A、B 都要解。多数"被觉得没产出"的情况，**70% 是 A（信息问题）**。

**二、解问题 A：信息不对称三动作**
1. **过程可见化**：别等季度末才汇报。设里程碑，主动同步——"这周完成了支付链路重构，故障率下降 X%"
2. **语言转译**：把技术语言翻译成业务价值。不说"重构了订单系统"，说"系统现在能扛 3 倍流量，大促不会崩"
3. **重定义产出口径**：主动拓宽评估维度，让"稳定性/效率/技术债清偿"也算产出，而不只是"上了几个新功能"

**三、向上汇报：把"约束"翻译成"决策题"**
❌ 别说："我们人手不够，做不快"（陈述句，让老板焦虑）
✅ 要说："当前架构支撑不了 XX 类需求，竞对已在布局，我们有约 2 个月窗口。要快，需要在【加人 / 砍需求 / 外部合作】里做选择——我的建议是 B，因为……"（判断题，让老板能拍板）

**公式**：现状约束 + 业务风险量化 + 时间窗口 + 可选方案 → 请决策层选。

**四、行动清单**
1. 本周起，建立"周度业务语言进展同步"机制（解 A）
2. 下次汇报用"决策题"公式重写一遍（解向上）
3. 若确有瓶颈，用"四类瓶颈诊断"定位是能力/资源/架构/需求哪一类，单独立项争取资源（解 B）

**理论来源**：信息不对称（Akerlof）、向上管理（Gabarro & Kotter, HBR 1980）、Drucker 沟通原则。
