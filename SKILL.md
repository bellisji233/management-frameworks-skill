## 使用方式：管理决策辅助

本 skill 用于管理决策辅助：**问题诊断 → 匹配 1–2 个理论 → 3 步内可操作的行动建议**。
触发信号："我遇到了…" / "怎么处理…" / "我的团队…"

## 诊断路由：先定位，再选框架

> ⚠️ 别一上来就堆框架。先用这个流程判断问题类型，再匹配 **1–2 个**最相关的框架。

```
管理问题进来
   │
   ├─ 是"个人/某个员工"的问题？
   │     ├─ 先看「能力 × 意愿」 → 单元二 情境领导力 3×3 矩阵（最高频入口）
   │     ├─ 涉及动力/积极性     → 单元三 五大激励理论（先判断断在哪）
   │     └─ 涉及绩效/难管员工   → 单元四 ABC 分层 + 三类高难度员工
   │
   ├─ 是"整个团队"的问题？
   │     ├─ 氛围/敬业度低 → 单元三 Q12（找最低分项）
   │     ├─ 接手新团队/绩效下滑溯源 → 单元一 麦肯锡 7S 扫描
   │     ├─ 新团队磨合/内讧/没人说真话 → 单元六 塔克曼阶段 / 心理安全感 / 五大障碍
   │     └─ 目标说不清/决策扯皮/没人拍板 → 单元七 OKR / RACI-RAPID
   │
   ├─ 是"组织/向上/跨部门"的问题？  → 单元五
   │     先分清：问题A 认知/信息问题  vs  问题B 资源/能力问题
   │     （两类不能混着解——光解释没用，光加速也没用）
   │
   ├─ 是"冲突/变革/招人"的具体场景？ → 单元八
   │     人际冲突 → TKI；推不动变革 → 科特8步/ADKAR；招错人/带新人 → 结构化招聘+30-60-90
   │
   └─ 是"我自己"的问题（精力/风格/自我激励）？
         时间精力分配 → 单元七 艾森豪威尔矩阵
         领导风格单一/带不出人 → 单元九 戈尔曼6风格 / 驱动力3.0
```

**框架选用要点**：个人问题看"能力 × 意愿"，团队问题看"敬业度 / 7S / 发展阶段"，向上/组织问题看"信息 vs 资源"，把事做成看"目标 / 决策 / 冲突"。

## 单元一：管理者效能（Manager Effectiveness）

### 管理天平
业务管理 ↔ 团队管理，随团队成熟度动态调整比重。

### 管理者5核心任务
*来源：Peter Drucker，《管理的实践》1954*
1. 制定目标
2. 组织工作
3. 激励与沟通
4. 培养人才
5. 绩效评估

### 高潜人才识别：10维度
*来源：DDI / 领导力发展领域实证研究综合*

领导意愿 · 价值观匹配 · 卓越绩效 · 正直真诚 · 培养他人 · 获得结果 · 影响他人 · 应变能力 · 学习能力 · 认知复杂度

> 📁 每个维度的观察信号详解见 `references/unit1-details.md`

### 麦肯锡7S诊断模型
*来源：Tom Peters & Robert Waterman，McKinsey，1980*

**硬性三角**（相对易变）：战略 Strategy · 结构 Structure · 制度 Systems

**软性四角**（根植文化，难变）：共同价值观 Shared Values · 风格 Style · 员工 Staff · 技能 Skills

使用场景：接手新团队快速扫描、绩效下滑溯源、变革阻力评估。

> ⚠️ **反模式**：7S 是"扫描诊断"工具，不是行动清单。别七个维度同时改——找到**最薄弱的那个 S** 先动。软性四角（价值观/风格/员工/技能）见效慢，急于求成往往事倍功半。

> 📁 14种管理行为详解见 `references/unit1-details.md`

## 单元二：授权、跟进与辅导（Delegation & Coaching）

### 执行质量四层次
做了（主动性）→ 做完（计划性）→ 做对（专业性）→ 做好（敬业性）

### 授权流程：3想 · 6选 · 5谈 · 3跟
- **3想**（授权前）：Why授权 · What授权 · How授权
- **6选**（选人维度）：能力 · 意愿 · 时间 · 资源 · 经验 · 信任
- **5谈**（授权对话）：WHAT任务 → WHY意义 → HOW方法 → ACTION确认 → SUPPORT承诺
- **3跟**（跟进方式）：定期汇报 · 关键节点检查 · 异常预警

### 情境领导力（Situational Leadership）× 差异化辅导
*来源：Paul Hersey & Ken Blanchard，1969；扩展为3×3矩阵*

```
        低能力          中能力          高能力
高意愿  ① 手把手教    ④ GROW辅导    ⑦ 充分授权
中意愿  ② 激发信心    ⑤ 引导自主    ⑧ 给予挑战
低意愿  ③ 坦诚对话    ⑥ 重塑动机    ⑨ 警示对话
```

**核心原则**：领导风格跟随员工发展阶段动态调整，不能用同一种方式管所有人。

### GROW教练模型
*来源：John Whitmore，《Coaching for Performance》1992*

Goal（目标）→ Reality（现状）→ Options（选择）→ Will（行动意愿）

用于辅导对话，少给答案，多用开放式提问引导员工自我发现。

> ⚠️ **反模式**：① 用同一种风格管所有人（最常见失效原因）；② 员工状态变了还用老画像——画像每季度更新一次；③ 对"低意愿"的人用 GROW 教练——动机问题不是辅导能解的，先解决意愿。

> 📁 九种情境详细操作方法见 `references/unit2-situational.md`

## 单元三：激励、沟通与团队建设（Motivation & Team Building）

### 五大激励理论速查

| 理论 | 来源 | 核心洞察 | 管理应用 |
|------|------|---------|---------|
| 需求层次理论 | Maslow, 1943 | 5层需求，低层满足后追求高层 | 先判断员工卡在哪一层 |
| 双因素理论 | Herzberg, 1959 | 保健因素≠激励因素 | 加薪减少抱怨，成长产生热情 |
| 强化理论 | Skinner, 行为主义 | 行为结果决定行为重复 | 及时具体的正向强化 |
| 公平理论 | Adams, 1963 | 比较投入产出比，非绝对值 | 解释决策依据，管理公平感知 |
| 期望理论 | Vroom, 1964 | 努力→绩效→奖励三链条 | 找断链：能力感知/兑现信任/价值契合 |

### 盖洛普Q12员工敬业度
*来源：Gallup，1990年代大规模实证研究，覆盖数百万员工*

12个问题分四层：基本需求 → 管理支持 → 团队归属 → 成长机会

使用方式：匿名调研1-5分，找团队最低分项优先改善。

> 📁 Q12完整12问见 `references/unit3-details.md`

### STAR反馈法
*来源：行为面试法（BEI），DDI等咨询公司推广*

Situation（背景）→ Task（任务）→ Action（行动）→ Result（结果）

应用：表扬、辅导、绩效面谈、向上汇报均适用。

> ⚠️ **反模式**：① 只靠加薪激励——薪资是保健因素，消除不满≠产生动力（双因素理论）；② 用同一种激励对所有人——先判断他卡在马斯洛哪一层、期望理论哪个链断了；③ 表扬泛泛而谈（"你很棒"）——强化要"及时+具体"才有效。

### 团队建设10种方法
知识类：读书会 · 案例研讨 · 经验萃取 · 轮岗 · 竞赛

文化类：兴趣社群 · 生日关怀 · 团队聚餐 · 入职仪式 · 运动活动

> 📁 详细说明见 `references/unit3-details.md`

## 单元四：绩效评估与发展面谈（Performance & Development）

### 绩效评估三维度
1. **业绩**：目标完成率，量化数据
2. **能力**：胜任力行为评估
3. **态度**：敬业度与工作作风

### ABC员工分层应对策略

| 类型 | 面谈策略 | 重点 |
|------|---------|------|
| A类（优秀） | 认可→自评→挑战→需求→展望 | 留住+给更大舞台 |
| B类（符合） | 自评→激励→提升路径→支持 | 稳住+推动成长 |
| C类（欠佳） | 坦诚→认可→诉求→改进→安排 | 明确边界+给机会 |

### 三类高难度员工处理（速查）
- **A 类提离职 · 留才 4 步**：真诚挽留 → 倾听诉求 → 分析判别（可/不可解决）→ 有条件接一接
- **B 类自评过高 · 3 步**：数据准备 → 引导其自我发现差距 → 共识行动计划
- **C 类持续抱怨 · 5 法**（基于 NVC）：倾听 → 视角转移 → 同理心回应 → 横向比较 → 升级对话

### 积极面谈法
充分准备 → 营造安全感 → 双向对话 → **聚焦未来** → 明确行动

> ⚠️ **反模式**：① 绩效面谈只谈过去、只算账（重点应是"未来怎么改"）；② 对 C 类只给压力不给路径；③ 因能力强就容忍态度问题（该"警示对话"还得上）。

> 📁 ABC 面谈话术、三类难管员工的详细步骤见 `references/unit4-details.md`

## 单元五：组织沟通与向上管理（Organizational Communication & Upward Management）

### 核心诊断：先分清两类问题
- **问题A·认知/信息**：外部/上级看不见你在做什么 → 信息透明化、主动管理预期
- **问题B·资源/能力**：真实的执行限制（技术债/人力/能力缺口）→ 显性化约束、向上争取资源

**关键原则**：两类不能混着解——光解释没用，光加速也没用。

### 五个框架（速查）
1. **信息不对称管理**（Akerlof）：过程可见化 + 语言转译（说业务价值）+ 重定义产出口径
2. **向上汇报**（Drucker + HBR）：把约束翻译成决策语言。**公式**＝现状约束 + 业务风险量化 + 时间窗口 + 可选方案 → 请决策层选
3. **瓶颈四类诊断**：能力缺口 / 资源不足 / 架构债务 / 需求不清——先定位再对症
4. **跨部门影响力**（Cohen & Bradford 1989）：无职权靠"货币交换"——把你的需求翻译成对方的收益
5. **向上管理**（Drucker；Gabarro & Kotter 1980）：不是拍马屁，是双向资源——理解上级目标 / 主动同步不让其意外 / 带方案去 / 管理期望

## 单元六：团队动力学（Team Dynamics）

> 从"管个人"到"管整个团队"。

- **塔克曼 4 阶段**（Tuckman 1965）：组建 → 震荡 → 规范 → 高效。每阶段管理者角色不同；**震荡期的冲突是正常且必经的**，不是团队坏了。
- **心理安全感**（Edmondson 1999；Google 验证）：成员敢冒人际风险，是高绩效团队第一要素。靠领导先示弱、把失败当学习、主动邀请异见来建立。
- **团队五大障碍**（Lencioni 2002）：信任缺失 → 惧怕冲突 → 欠缺投入 → 逃避责任 → 无视结果。**自下而上建设，先建信任。**

> ⚠️ **反模式**：把"震荡期冲突 / 没人反对"当成团队没问题；追求表面和谐、回避一切冲突。

> 📁 阶段对照表、建立动作与职场案例见 `references/unit6-details.md`

## 单元七：目标、决策与执行（Goals · Decisions · Execution）

> 从"带人"到"把事做成"。

- **OKR**（Grove → Doerr）：O 鼓舞人心定性 + KR 可量化 3–5 个。OKR ≠ KPI（OKR 0.7 即优秀、导向对齐；KPI 要求 100%、导向考核）。
- **RACI / RAPID**（RAPID 来自 Bain）：治决策扯皮。核心是**拍板人唯一**（RACI 的 A / RAPID 的 D 必须只有一个）。
- **艾森豪威尔矩阵**：重要×紧急四象限。管理者最该刻意投资**"重要不紧急"**（战略/培养人/建机制），它最易被紧急挤掉。

> ⚠️ **反模式**：把 OKR 当 KPI 考核、KR 不可量化；拍板人多于一个；被"紧急"绑架、忽视"重要不紧急"。

## 单元八：冲突、变革与选才（Conflict · Change · Hiring）

> 高频刚需场景。

- **TKI 冲突模式**（Thomas & Kilmann 1974）：果断×合作分 5 种——竞争 / 协作⭐ / 回避 / 迁就 / 妥协。**没有最好的，看情境选；关键是因情境切换，不要只会一种。**
- **变革管理**（Kotter 1996 + Prosci ADKAR）：科特 8 步管"组织怎么变"，ADKAR（认知→意愿→知识→能力→巩固）管"个人怎么变"。推不动时用 ADKAR 查卡在哪关，多数卡在"意愿"。
- **结构化招聘 + 30-60-90**：同岗同题同评分表；**以行为题为主、假设题为辅**（STAR）；多面试官独立打分。入职后 30 天学习 → 60 天小步贡献 → 90 天独立产出。

> ⚠️ **反模式**：冲突永远用同一种；以为"发了通知"变革就发生、不设计短期胜利；招人凭"聊得来"、新人放任不管。

> 📁 TKI 看图方法、ADKAR 诊断、面试与 onboarding 细节及职场案例见 `references/unit8-details.md`

## 单元九：领导力进阶（Advanced Leadership）

> 关于"你自己"——管理者的自我领导力。

- **戈尔曼 6 种领导风格**（Goleman, HBR 2000）：远见 / 教练 / 亲和 / 民主（通常正面）+ 领跑 / 命令（慎用）。**高手随情境切换**——危机用命令型止损，带人用教练型。
- **驱动力 3.0**（Pink 2009）：对知识/创造性工作，钱过某点失效甚至有害。内在动力 = **自主 + 精通 + 目的**，与期望理论"价值契合"相通。

> ⚠️ **反模式**：常年只用一种风格（尤其"领跑型"累垮团队）；对需内驱的创造性工作用纯金钱/KPI 强激励。

## 使用规范

### 管理决策辅助模式
1. **走诊断路由**（见上），识别核心矛盾类型（个人/团队/组织）
2. 判断是"认知问题"还是"资源问题"（或两者都有）
3. 匹配 **1-2 个**最相关框架（别堆框架）
4. 输出：问题诊断 → 推荐理论 → **3步内可操作**行动建议
5. **给效果验证**：每个建议附"怎么判断奏效了"的可观察信号 + 复查时间点
   （如"30天后看主动性是否回来""下次汇报老板是否能直接拍板"）
6. 标注理论来源，区分"成熟理论"与"实践框架"
